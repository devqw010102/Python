# 🌐 OSI 7계층 모델 정리 (OSI 7 Layer Model)
- ISO 에서 개발
- 각 계층은 독립적으로 구성, 상위 계층의 기능은 하위 계층의 기능을 이용
- 7가지 계층으로 정의
- 계층 구성
   1. 하위 계층 : 데이터를 목적지에 정확하게 전달하는 것에 초점
      - **물리 계층 → 데이터 링크 계층 -> 네트워크 계층**
   2. 상위 계층 : 데이터의 가공, 활용에 초점
      - **전송 계층 -> 세션 계층 -> 표현 계층 -> 응답 계층** 

---

## 🏗️ OSI 7계층 계통도

| 계층    | 이름                     | 데이터 단위            | 핵심 역할              | 주요 프로토콜              | 장비           |
|:------|:-----------------------|:------------------|:-------------------|:---------------------|:-------------|
| **1** | **물리 (Physical)**      | Bit               | 전기적 신호 전송          | RS-232C              | 허브, 리피터, 케이블 |
| **2** | **데이터 링크 (Data Link)** | Frame             | 물리 주소(MAC) 기반 통신   | 이더넷                  | 스위치, 브리지     |
| **3** | **네트워크 (Network)**     | Packet            | 논리 주소(IP) 기반 경로 설정 | IP, ICMP             | 라우터          |
| **4** | **전송 (Transport)**     | Segment, Datagram | 종단 간 신뢰성/흐름 제어     | TCP, UDP             | 게이트 웨이       |
| **5** | **세션 (Session)**       | Data              | 통신 연결 유지 및 동기화     | SSH, TLS, RPC        |              |
| **6** | **표현 (Presentation)**  | Data              | 데이터 형식 변환, 암호화, 압축 | JPEG, MPEG, ASCII    |              |
| **7** | **응용 (Application)**   | Data              | 사용자 서비스 인터페이스      | HTTP, FTP, SMTP, DNS |              |

---

## 📝 계층별 상세 설명

### [L1] 물리 계층 (Physical Layer)
- **설명**: 데이터를 전기적, 기계적 신호로 변환하여 실제 전송 매체를 통해 주고받는 최하위 계층입니다.
- **특징**: 주소 개념이 없으며 단순히 **비트(0, 1)**를 전송하는 데 집중합니다.

### [L2] 데이터 링크 계층 (Data Link Layer)
- **설명**: 물리적인 네트워크 사이의 데이터 전송을 신뢰성 있게 보장합니다.
- **특징**: 랜카드에 적힌 **MAC 주소**를 사용하여 통신하며, 에러 검출(CRC)과 흐름 제어를 수행합니다.

### [L3] 네트워크 계층 (Network Layer)
- **설명**: 데이터를 목적지까지 가장 안전하고 빠르게 전달하기 위한 경로(Routing)를 설정합니다.
- **특징**: **IP 주소**라는 논리적 주소를 사용하며, 패킷 단위로 데이터를 쪼개어 전달합니다.



### [L4] 전송 계층 (Transport Layer)
- **설명**: 양 끝단(End-to-End)의 사용자들이 신뢰성 있는 데이터를 주고받을 수 있도록 보장합니다.
- **특징**: 데이터가 제대로 도착했는지 확인하며, **포트(Port) 번호**를 통해 상위 프로세스를 구분합니다.

### [L5] 세션 계층 (Session Layer)
- **설명**: 응용 프로그램 간의 대화를 관리하고 동기화합니다.
- **특징**: 통신 연결(세션)의 설정, 유지, 종료를 담당하며 체크포인트를 두어 통신 장애 시 복구를 돕습니다.

### [L6] 표현 계층 (Presentation Layer)
- **설명**: 수신자가 읽을 수 있는 형식으로 데이터를 인코딩하거나 암호화합니다.
- **특징**: 서로 다른 데이터 표현 방식(예: 확장자, 문자 인코딩)을 일치시키고, **압축 및 암호화/복호화**를 수행합니다.

### [L7] 응용 계층 (Application Layer)
- **설명**: 사용자가 네트워크 서비스에 접근할 수 있는 인터페이스를 제공하는 최상위 계층입니다.
- **특징**: 우리가 흔히 쓰는 웹 서핑(HTTP), 파일 전송(FTP), 메일(SMTP) 등의 서비스가 이 계층에서 작동합니다.



---

## ⚙️ 주요 개념
* **캡슐화 (Encapsulation)**: 데이터가 상위에서 하위 계층으로 내려가며 각 계층의 헤더 정보가 붙는 과정입니다.
* **역캡슐화 (Decapsulation)**: 수신 측에서 하위에서 상위 계층으로 올라가며 헤더 정보를 제거하고 원본 데이터를 복원하는 과정입니다.

# 📡 프로토콜
- 정보 전달 시에 약속한 규칙

### 🔑 네트워크 프로토콜의 3요소

프로토콜(Protocol)이란 서로 다른 기기 간에 데이터를 원활하게 주고받기 위해 정의된 **통신 규약**을 의미하며, 아래의 세 가지 기본 요소로 구성됩니다.

| 기본 요소              | 설명                                                              | 핵심 키워드       |
|:-------------------|:----------------------------------------------------------------|:-------------|
| **구문 (Syntax)**    | 전송하고자 하는 데이터의 **형식(Format)**, **부호화(Coding)**, **신호 레벨** 등을 규정  | 형식, 구조, 부호화  |
| **의미 (Semantics)** | 두 기기 간의 효율적이고 정확한 정보 전송을 위한 **전송 제어**, **제어 정보** 및 **에러 관리** 규정 | 제어 정보, 오류 제어 |
| **타이밍 (Timing)**   | 두 기기 간의 통신 **속도 조절**이나 메시지의 **전송 순서** 등을 규정                     | 속도 조절, 순서 제어 |

### 🏘️ 네트워크 규모(범위)에 따른 분류

네트워크가 커버하는 지리적 범위에 따라 아래와 같이 세 가지로 분류합니다.

| 분류      | 명칭                                  | 범위                | 특징                               |
|:--------|:------------------------------------|:------------------|:---------------------------------|
| **LAN** | 근거리 통신망 (Local Area Network)        | 건물, 학교, 사무실 내     | 고속 전송이 가능하며 에러 발생률이 매우 낮음        |
| **MAN** | 도시권 통신망 (Metropolitan Area Network) | 도시 단위 (약 50km 이내) | LAN과 WAN의 중간 형태, 도시 내 여러 LAN을 연결 |
| **WAN** | 광역 통신망 (Wide Area Network)          | 국가, 대륙, 전 세계      | 전송 거리가 매우 멀고 에러율이 높으며 라우팅 기술 필수  |

---

>1. **규모 크기 순서:** LAN < MAN < WAN
>2. **LAN (근거리 통신망):**
>   - 동일 구내, 동일 건물 내의 장치들을 연결.
>   - 자원 공유를 목적으로 함.
>   - 단일 기관이 소유하며 전송 속도가 빠름.
>3. **WAN (광역 통신망):**
>   - 국가와 국가, 대륙과 대륙을 연결하는 광범위한 네트워크.
>   - 통신 회선 업체의 회선을 빌려서 사용 (공중망 사용).



## 층별 핵심 프로토콜 요약

### 1. 응용 계층 (Application Layer) - 서비스 제공
사용자가 직접 사용하는 서비스와 관련된 프로토콜입니다.

* **HTTP (HyperText Transfer Protocol)**: 웹 페이지(HTML)를 주고받기 위한 프로토콜.
* **FTP (File Transfer Protocol)**: 컴퓨터 간 파일을 전송하기 위한 프로토콜.
* **SMTP (Simple Mail Transfer Protocol)**: 이메일을 **보낼 때** 사용.
* **POP3 (Post Office Protocol v3)**: 이메일을 **서버에서 가져올 때** 사용.
* **SNMP (Simple Network Management Protocol)**: 네트워크 장비를 관리하고 감시하기 위한 프로토콜.
* **DNS (Domain Name System)**: 도메인 네임(www.google.com)을 IP 주소로 변환.
* **TELNET**: 멀리 떨어진 컴퓨터에 접속하여 자신의 컴퓨터처럼 사용할 수 있는 원격 접속 서비스.



---

### 2. 전송 계층 (Transport Layer) - 신뢰성 및 흐름 제어
데이터를 어떻게 운반할지 결정하는 계층입니다.

* **TCP (Transmission Control Protocol)**
    * **연결 지향형**: 송수신 측을 연결한 후 데이터 전송.
    * **신뢰성**: 흐름 제어, 오류 제어를 통해 정확한 전송 보장.
    * **양방향(Full Duplex)** 통신.
* **UDP (User Datagram Protocol)**
    * **비연결형**: 연결 없이 데이터를 일방적으로 보냄.
    * **속도 중심**: 확인 응답이 없어 TCP보다 빠르지만 신뢰성이 낮음 (실시간 스트리밍에 사용).



---

### 3. 네트워크 계층 (Network Layer) - 경로 및 주소 설정
목적지까지의 최적 경로(Routing)를 찾는 계층입니다.

* **IP (Internet Protocol)**: 패킷을 목적지 주소로 전달하는 핵심 프로토콜.
* **ICMP (Internet Control Message Protocol)**: 네트워크 내 에러 보고 및 진단 (예: `ping` 명령 시 사용).
* **ARP (Address Resolution Protocol)**: **IP 주소**를 알고 있을 때 **MAC 주소**를 알아내는 프로토콜.
* **RARP (Reverse ARP)**: **MAC 주소**를 알고 있을 때 **IP 주소**를 알아내는 프로토콜.

---

### 4. 데이터 링크 계층 (Data Link Layer) - 직접적인 데이터 전송
인접한 노드 간의 물리적인 연결을 관리합니다.

* **Ethernet (이더넷)**: 가장 대표적인 유선 LAN 표준 프로토콜.
* **HDLC**: 비트 위주의 고속 데이터 전송 프로토콜.
* **PPP**: 두 지점 간(Point-to-Point) 직접 연결하는 프로토콜.

---

## 💡 키워드 암기

1.  **메일 송신** = SMTP / **메일 수신** = POP3
2.  **IP ➔ MAC** = ARP / **MAC ➔ IP** = RARP
3.  **신뢰성/연결형** = TCP / **비신뢰성/속도** = UDP
4.  **원격 접속** = TELNET
5.  **에러 확인/Ping** = ICMP
